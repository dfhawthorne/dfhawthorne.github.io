Overview

When Grid Infrastructure is initially installed, only one (1) ASM disk group is created (usually DATA ). This procedure shows how to add an extra one using asmca .

References

None

Procedure

Invoke ASMCA

First, set the ORACLE_HOME to the GI Home binaries. Then, run: 

asmca

Configure ASM: Disk Groups

The following screen then appears:


Click Create .

Create Disk Group

On the following screen, enter the following options:

  • Disk Group Name is FRA
  • Redundancy is External because this is a home system, and I like living dangerously
  • Select /dev/sdc

NOTE : If there are no candidate volumes, check that the devices are owned by the oracle user.


Click OK .

The following message then appears and stays for several minutes:


Then the confirmation messages appears:


Click OK .

Configure ASM: Disk Groups

The new disk group ( FRA ) appears in the list of disk groups.


Click Exit .

Then the confirmation messages appears:


Click Yes .
