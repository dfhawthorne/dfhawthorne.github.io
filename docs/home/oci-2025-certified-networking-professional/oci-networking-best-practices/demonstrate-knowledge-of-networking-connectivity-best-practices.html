Fully-Redundant Network Connectivity

A fully-redundant network connectivity is needed to preserve business continuity, and to ensure availability of applications and workloads during both planned and unplanned outages.

Inspection of the network design should reveal no single points of failure (SPOF). Fastconnects are not redundant if they terminate at the same POP or edge router. An Oracle FastConnect partner is required to have redundant connections, thereby alleviating the customer from verifying such.

Network redundnacy should be verified with regular failover tests.

Network Connectivity: Best Practice

Carefully review the trade-offs between FastConnect and VPN with respect to performance and costs.

Although auto-generated DRG tables and import route distributions are fine for the most basic scenarios, custom ones should be created for more complex DRG scenarios in order to match the requirements.
