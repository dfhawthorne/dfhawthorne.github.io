Modifications

Using the original procedure, I found it difficult to install VMware Tools correctly. I needed to run the VMware Toolbox so that the timing problem noted in NTP Saga Ends is resolved by setting the Time Synchronization as show below:


The primary modification to the procedure is in the choice of packages initially loaded when creating the OS image:

  • Desktop Environments
    • GNOME Desktop Environment
  • Base System
    • Administration Tools
    • Base
    • X Window System

Procedure

Using RHEL 5.4 DVD,

  1. Installing using GUI.
  2. Skipped CD Media test.
  3. Installation language is English (English) .
  4. Keyboard is US English .
  5. Skip entering Installation Number.
  6. Confirm skip.
  7. Confirm initialisation of disk drive to erase all data.
  8. Remove all partitions on selected drives and create default layout.
  9. Confirm.
  10. Click Next on disk layout.
  11. Accept the defaults for the boot loader.
  12. Network Device configuration:
    1. Edit eth0 :
      • Is active on boot.
      • Enable IPv4 support.
      • Manual configuration is 10.1.1.240/24
      • Disable IPv6 support.
    2. Edit eth1 :
      • Is active on boot.
      • Enable IPv4 support.
      • Manual configuration is 192.168.1.1/24
      • Disable IPv6 support.
    3. Edit eth2 :
      • Is active on boot.
      • Enable IPv4 support.
      • Manual configuration is 192.168.2.2/24
      • Disable IPv6 support.
    4. Hostnane is set manually to penrith1.yaocm.id.au
    5. Gateway is 10.1.1.240
    6. Primary DNS is 10.1.1.252 (GRIDCTRL)
    7. Secondary DNS is 10.1.1.1 (ADSL Router)
  13. Set location to Australia/Sydney with System clock uses UTC .
  14. Set the root password to ******** .
  15. For software selection, choose Customize now .
  16. Package selection ( See above for modified list ):
    • Under Base System , leave Base as the only selected option.
    • There are no items selected in any of the other options.
  17. Start the installation.
