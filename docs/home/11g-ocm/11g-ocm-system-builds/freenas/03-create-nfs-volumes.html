FreeNAS TM 8.0.2 USERS GUIDE

Configuring Oracle Clusterware Storage

Create Groups and Users

Create the groups and users as specified in 04 Oracle Clusterware Preinstallation Tasks through the Web GUI at http://10.1.1.239 .

Create Volumes

Create Volume

Use the Web GUI at http://10.1.1.239

  1. Open Storage Menu
  2. Open Volumes Menu
  3. Select Volume
    1. Set Volume name to PENRITH .
    2. Select member disks to da0 (21.5 GB) .
    3. Select ZFS as the Filesystem type .
    4. Leave Force 4096 sector size unchecked.

Create ZFS Datasets

The ZFS Volume will now be partitioned up into separate filesystems by creating ZFS datasets (using the sizes recommended in Requirements for Using a File System for Oracle Clusterware Files ):

  1. Click Add ZFS Dataset .
  2. Create ZFS datasets for the following of size 280 MB for all four (4) entry fields:
    • OCR1
    • OCR2
    • VOTE1
    • VOTE2
    • VOTE3
  3. Change the permissions as follows:
    1. Set Owner (user) to crs
    2. Set Owner (group) to oinstall
    3. Leave Mode as is
    4. Leave Type of ACL unchecked
    5. Leave Set permission recursively unchecked

Create NFS Shares

  1. Open Sharing Menu
  2. Open NFS Shares
  3. Select Add NFS Share
    1. Set Comment to Penrith Cluster OCR #1
    2. Set Path to /mnt/PENRITH/OCR1
    3. Set Authorized network or IP addresses to 192.168.2.0/24
    4. Leave the following unchecked:
      • All Directories
      • Read Only
      • Quiet
    5. Set Maproot User to root
    6. Leave the following selected as N/A
      • Maproot Group
      • Mapall User
      • Mapall Group
  4. Repeat this for the other ZFS datasets:
    • /mnt/PENRITH/OCR2
    • /mnt/PENRITH/VOTE1
    • /mnt/PENRITH/VOTE2
    • /mnt/PENRITH/VOTE3

Enable NFS

The NFS service must be turned on via Control Services under Services in the Web GUI.
