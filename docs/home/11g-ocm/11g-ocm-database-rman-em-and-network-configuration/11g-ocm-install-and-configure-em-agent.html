Overview

I was able to use both the OEM Agent Deploy Application and the agentDeploy script but only for 10.2.0.1.0 installation. The 10.2.0.5.0 files on the OMS server were not set up correctly for agent deployment. I had to download the 3731593 patch files I had saved on GRIDCTRL in order to use OUI to upgrade the EM agent to 10.2.0.5.0.

Points to Remember

OEM Agent Deploy Application

Uner RHEL 5.4, the sudo feature is not available because sudo requires a tty and is not available from ssh .

Best to supply the management password on the deployment, else you will have to separately secure the agent.
