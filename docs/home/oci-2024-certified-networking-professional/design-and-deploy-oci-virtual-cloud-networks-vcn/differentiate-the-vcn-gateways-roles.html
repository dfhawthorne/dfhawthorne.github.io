Summary

There are at least five (5) VCN Gateway roles:

  1. DRG: Dynamic Routing Gateway is used to route between the private on-premise network.
  2. Internet: Internet Gateway is used to route between a VCN and the Internet.
  3. LPG: Local Peering Gateway is used to route between VCNs in the same region.
  4. NAT: An optional virtual router that you can add to your virtual cloud network (VCN) to perform Network Address Translation (NAT). A NAT gateway gives cloud resources without public IP addresses access to the internet without exposing those resources to incoming internet connections.
  5. Service: An optional virtual router that you can add to your virtual cloud network (VCN). The gateway enables on-premises hosts or VCN hosts to privately access Oracle services (such as Object Storage and Autonomous Database) without exposing the resources to the public internet.
