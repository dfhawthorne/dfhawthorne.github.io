Summary

The optimal network configuration is a hub-and-spoke design with a FastConnect backbone for the link between on-premises and OCI. Migration to OCI encompasses moving large amounts of data to either block volumes or object storage.

Network Overview

The following diagram shows four (4) connection methods:

  1. Data Transfer Appliance
  2. Internet connection to Oracle SaaS
  3. FastConnect Public Peering to Oracle Edge
  4. FastConnect Private Peering to an OCI region

Design Criteria

The following design criteria need to be considered:

Connection redundancy
Whether there are multiple FastConnect links, or site-to-site VPN as backup
Load Balancing
Use OCI load balancing accross links to improve resource utilisation, allow scaling, and allow high availability
Bypass the internet for public resources
Implement FastConnect with public peering to access public OCI services without using the internet, or use a service gateway to allow cloud resources without public IP addresses to privately access Oracle services.
