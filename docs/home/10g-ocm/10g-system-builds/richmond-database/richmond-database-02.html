Started installing the database on richmond1 and richmond2 . Found one (1) problem during the pre-installation checks: the shmmax parameter was too small on richmond1 but was correct on richmond2 . This was easily fixed by editting /etc/sysctl.conf and sysctl -p .

ORA-15032: not all alterations performed
ORA-15063: ASM discovered an insufficient number of disks for diskgroup "DATA"
