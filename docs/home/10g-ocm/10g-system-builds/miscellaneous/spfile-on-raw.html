When I created the ASM database on the padstow cluster, I defined the spfile to be on the /dev/raw/raw3 device.

This appearred to work despite the ownership being root:oinstall with permissions 660. Looks like oracle was able to write to the device okay via its group membership.

I changed the ownership anyway to oracle:dba to be consistent with the ASM devices.

The size of the /dev/raw/raw3 device is 256M (same as the OCR disk).
