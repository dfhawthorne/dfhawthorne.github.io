Thursday 01 May, 2008 - 11:05

I have run the pre-installation check for ClusterWare for the OCR to be on /dev/sdb1 and the voting disk on /dev/sdd1 :
$CV_HOME/bin/cluvfy stage -pre crsinst -c /dev/sdb1 -o /dev/sdd1 -verbose

The problems found are:

  • VIP address (noted in CLUVFY NODECON (2) )
  • The shared disks (noted in CLUVFY SSA (1) )
  • Memory size should be at least 512 Mb (currently 256Mb)
  • Swap space should be 1Gb

Decided to rebuild the RHEL3.7 using a swap partition (see Updating RHEL4.2 for Or*cle ) and to put /tmp as a separate partition.
