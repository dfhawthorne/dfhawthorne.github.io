The problem described in Richmond Cluster (22) was caused by my adding a new PCI card to the richmond2 image and not using kudzu to configure the card.

The strange thing is that crs worked fine without having access to the ocr or voting disks.

So the output from the ocrcheck command was correct but misleading. The underlying problem was that the raw files were on an unconfigured PCI card.
