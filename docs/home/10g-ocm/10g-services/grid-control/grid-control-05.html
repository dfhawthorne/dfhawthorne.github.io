The problem described in Grid Control (04) was caused by the OMS not running on gridctrl after a restart.

When I started the Oracle instance (emrep), Oracle listener (LISTENER), and the OMS service, I was able to acces the OMS page.

The root.sh script in OMS_HOME updates the /etc/gcstartup which only gets called when the runlevel is set to 3. (I found the solution at Thread: auto startup script in 10g rel2 )

Editted /etc/inittab to change the default runlevel and rebooted.

Except there is now a problem with the Virtual LAN and my MetaLink account is expired!
