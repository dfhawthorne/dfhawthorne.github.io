I have managed to get around the startup problems with gridctrl temporarily by starting the systems individually and waiting for initialisation to complete.

As such, I am able to run all four (4) systems satisfactorily at this stage.

To the OMS agent to start on richmond1 and richmond2 , I had to set the default runlevel to 3 via /etc/inittab . This would be a consistent design approach by Oracle to have all managed servers running without X11 in multiuser mode.
