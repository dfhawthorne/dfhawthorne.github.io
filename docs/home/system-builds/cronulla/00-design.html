Design

Choice of Installation Type

Since I am now installing OEM on a separate server, I would like to try an advanced installation (SMALL) as described in 8.1.1 Overview of the Installation Types .

Prerequisites

The requirements for a successful installation are listed in 8.3 Prerequisites .

Patch Set Requirements

MOS Doc ID 793512.1 says that EM12103 is the latest release for Linux x86 (64-bit).

MOS Doc ID 822485.1 says that Patch 17513525 (OMS Generic) is the latest PSU available for this release (EM12103).

Hardware Requirements

Since I have chosen advanced installation (SMALL) , the hardware requirements as described in 4.1 CPU, RAM, Heap Size, and Hard Disk Space Requirements for Oracle Management Service are:

Component Size
CPU 2
RAM 6GB
Disk Space 15GB

Note: I do plan to use either of the following options:

  1. ADP Manager is Application Dependency and Performance
  2. JVMD is JVM Diagnostics

Operating System

I plan to use Oracle Enterprise Linux 6.5 (OEL6u5 - 64bit) to build the OMS host.

Package Requirements for Oracle Management Service

Since I am planning to use OEL6u5 (64-bit) to build the server, I would need the following packages (according to 5.1.3 Package Requirements for Oracle Management Service ):

  • make-3.81
  • binutils-2.20.51.0.2-5.11
  • gcc-4.4.4
  • libaio-0.3.107
  • glibc-common-2.12-1
  • libstdc++-4.4.4
  • libXtst-1.0.99.2-3.el6.x86_64.rpm
  • sysstat-9.0.4
  • glibc-devel-2.12-1.7.el6.i686 (This is a 32-bit package)
  • glibc-devel-2.12-1.7.el6.x86_64 (This is a 64-bit package)

Kernel Parameters

Since the OMS and OMR will be on different Linux hosts, the advice given in 5.2.1 Kernel Parameter Requirements for Oracle Management Service is:

If you are installing OMS and Management Repository (database) on different hosts, which are running on Linux operating systems (32-bit or 64-bit), then on the OMS host, set the kernel.shmmax parameter to a value 1 byte less than 4 GB or 4294967295.

Library Requirements

Since I am planning to use OEL6u5 (64-bit), the advice given in 5.3.1 Library Requirements for Oracle Management Service is to use the following library:

  • glibc-2.5.12
