Overview

In order to fix the problem raised in Unable to Connect to OMS on CRONULLA , repvfy was installed in REPOS .

References

  • EMDIAG REPVFY Kit for Cloud Control 12c - Download, Install/De-Install and Upgrade (Doc ID 1426973.1)
  • Download EMDIAG Repvfy Kit Release for EM 12c Repository
  • EMDIAG Repvfy 12c Kit - How to Use the Repvfy 12c kit (Doc ID 1427365.1)

Procedure

Download the EMDIAG Repvfy Kit

repvfy is downloaded from Download EMDIAG Repvfy Kit Release for EM 12c Repository into /opt/oracle/software .

Install the EMDIAG Repvfy Kit

Following the procedure in EMDIAG REPVFY Kit for Cloud Control 12c - Download, Install/De-Install and Upgrade (Doc ID 1426973.1) , unzip the downloaded file as follows: 

cd /opt/oracle/software
unzip -q repvfy12_2014.0403.zip -d ${ORACLE_HOME}/emdiag

Checked that the repvfy script is executable: 

[oracle@gridctrl software]$ ls -l ${ORACLE_HOME}/emdiag/bin/repvfy
-rwx------ 1 oracle oinstall 7362 Apr  4 07:15 /opt/oracle/app/OracleHomes/db11g/emdiag/bin/repvfy

Installed the repvfy kit as follows: 

cd ${ORACLE_HOME}/emdiag/bin
./repvfy install -tns "(DESCRIPTION=(ADDRESS=(PROTOCOL=tcp)(HOST=gridctrl.yaocm.id.au)(PORT=1521))(CONNECT_DATA=(SID=repos)))"
